# HTTP API 契約：POST /api/v1/upload-agent

**Endpoint**: `POST /api/v1/upload-agent`
**實作版本**: CyberBrick OTA Agent v1.4.0+
**用途**: 無線更新裝置上的 OTA agent Python 原始碼，完成後裝置自動重啟（FR-005）

## Security Notice

> ⚠️ **安全**：Request body 包含 OTA token、WiFi 密碼等 credentials（由 `buildCyberBrickOtaAgentSource()` 嵌入）。
> **絕對不能**將 body 內容記錄至任何 log 系統或顯示於 UI 中。

## Request

### Headers（必要）

| Header | 格式 | 說明 |
|--------|------|------|
| `Authorization` | `Bearer {otaToken}` | OTA Token（當前裝置的 token）|
| `X-CyberBrick-Device-ID` | `{deviceId}` | 裝置識別碼 |
| `X-CyberBrick-Protocol-Version` | `"2"` | OTA 通訊協定版本 |
| `Content-Type` | `text/plain` | Agent 原始碼為純文字 |
| `Content-Length` | `{bytes}` | Body 位元組數（UTF-8 編碼）|
| `X-Singular-Content-Sha256` | `{hex}` | Body 的 SHA-256 hex digest（完整性驗證）|

### Body

新版 MicroPython agent 的完整原始碼（`cyberbrickOtaAgentSource.ts` 中 `buildCyberBrickOtaAgentSource(config)` 的輸出），UTF-8 編碼。

包含以下嵌入值（均已是裝置本身的 credentials，非新 credentials）：
- `DEVICE_ID`：裝置識別碼
- `OTA_TOKEN`：當前 OTA token
- `OTA_PORT`：OTA 伺服器埠
- `WIFI_SSID` / `WIFI_PASSWORD`：WiFi 設定

### 範例 Request

```http
POST /api/v1/upload-agent HTTP/1.1
Host: 192.168.1.100:8266
Authorization: Bearer abc123def456
X-CyberBrick-Device-ID: cb-device-uuid-001
X-CyberBrick-Protocol-Version: 2
Content-Type: text/plain
Content-Length: 12345
X-Singular-Content-Sha256: d2a84f4b8b650937ec8f73cd8be2c74add5a911ba8f0bb4f5fd7b85e3e57bd9f

# CyberBrick OTA agent v1.5.8
# Generated by Singular Blockly.
...（Python 原始碼 — 不記錄此內容）
```

## Response

### 成功（200 OK）

> 回應 body 為 JSON；目前裝置端實作**不保證**附帶 `Content-Type: application/json` header。

```json
{
  "contentSha256": "d2a84f4b8b650937ec8f73cd8be2c74add5a911ba8f0bb4f5fd7b85e3e57bd9f"
}
```

| 欄位 | 型別 | 說明 |
|------|------|------|
| `contentSha256` | `string` | 裝置計算的已寫入內容 SHA-256（Extension Host 可選驗證）|

### 完整性驗證失敗（400 Bad Request）

```json
{
  "error": "write",
  "contentSha256": "xxxxxx..."
}
```

### 驗證失敗（401 Unauthorized）

```json
{ "error": "auth" }
```

### Agent 版本不支援（404 Not Found）

當 Agent < 1.4.0 時，此 endpoint 不存在，Extension Host 以此判斷無法無線升級（FR-008）。

## 行為規格

### 驗證順序

1. Authorization token 驗證
2. X-CyberBrick-Device-ID 驗證
3. X-CyberBrick-Protocol-Version 驗證
4. SHA-256 完整性驗證（接收完整 body 後）

### 寫入與重啟

1. 接收並驗證完整 body
2. 計算 body SHA-256，與 `X-Singular-Content-Sha256` 比對
3. 寫入 `cyberbrick_ota_agent.py`（固定路徑）
4. 回傳 200 OK 與 `contentSha256`
5. 將 reboot flag 標記為待處理；request 完成後短暫等待（約 200ms）再呼叫 `machine.reset()`

### 重啟等待（Extension Host 端）

Extension Host 在收到 200 OK 後，輪詢 `GET /api/v1/health` 等待裝置就緒：

```
回應 200 OK
  → 等待 1s（裝置開始重啟）
  → 輪詢 health（1s 間隔）
  → 就緒（status: 'ready'）→ 繼續上傳程式碼（FR-007）
  → 逾時 30s → 顯示警告，繼續上傳（FR-007a）
```

### 進度提示（Extension Host 端，FR-006）

```
觸發 upgrading_agent stage：
  → i18n: OTA_AGENT_UPGRADING（含 {currentVersion} → {targetVersion}）

升級成功後觸發 agent_upgraded stage：
  → i18n: OTA_AGENT_UPGRADED（含 {targetVersion}）

升級失敗後觸發警告（非 failed stage）：
  → 繼續執行後續上傳流程
```

## MicroPython 裝置端實作

```python
# POST /api/v1/upload-agent
def _handle_upload_agent(client, headers, leftover):
  expected = headers.get('x-singular-content-sha256', '')
  _, actual = _write_uploaded_file(client, leftover, int(headers.get('content-length', 0)), 'cyberbrick_ota_agent.py')
  if actual != expected:
    return _send_json(client, 400, {'error': 'write', 'contentSha256': actual})
  reboot_pending[0] = True
  return _send_json(client, 200, {'contentSha256': actual})
```

## Extension Host 實作（偽程式碼）

```typescript
// cyberbrickOtaUploader.ts — upgradeAgentOverWifi()
private async upgradeAgentOverWifi(
    device: PairedCyberBrickDevice,
    token: string,
    config: AgentBuildConfig
): Promise<'upgraded' | 'failed'> {
    const agentSource = buildCyberBrickOtaAgentSource(config);
    const agentBytes = Buffer.from(agentSource, 'utf8');
    const sha256 = crypto.createHash('sha256').update(agentBytes).digest('hex');

    const resp = await this.fetchImpl(`${this.baseUrl(device)}/api/v1/upload-agent`, {
        method: 'POST',
        headers: {
            ...this.buildHeaders(device.deviceId, token),
            'Content-Type': 'text/plain',
            'Content-Length': String(agentBytes.length),
            'X-Singular-Content-Sha256': sha256,
        },
        body: agentBytes,
        signal: AbortSignal.timeout(this.uploadTimeoutMs),
    });

    if (!resp.ok) return 'failed';

    const ready = await this.waitForAgentReady(device, token);
    return ready ? 'upgraded' : 'failed';  // FR-007a：超時仍視為 'failed'，但繼續上傳
}
```
